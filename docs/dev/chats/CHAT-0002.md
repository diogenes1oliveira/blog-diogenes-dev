### Message #1 — Agent

The real culprit was `.editorconfig`, not markdownlint. The global `[*]` section had `trim_trailing_whitespace = true`, and the `[*.md]` section did not override it — so VS Code was stripping the two trailing spaces at save time, before markdownlint ever ran. Added `trim_trailing_whitespace = false` to the `[*.md]` section to override the global rule for markdown files only. Also restored the trailing spaces in the poem file.
